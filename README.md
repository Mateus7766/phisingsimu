# Tema

Protótipo de Sistema Computacional

# Questionamentos

## •	Quais problemas o sistema simula/resolve?

O sistema simula um ataque de phishing em ambiente controlado onde o "golpista" cria um site similar á uma empresa já existente e realiza práticas ilegais onde pode roubar dados sensíveis como: emails, senhas, número de cartões e outros documentos importantes da vítima. 

## •	Quais conceitos de Sistemas Computacionais e Segurança foram aplicados?

Dentro do site há outras funcionalidades como conversão numérica e criptografia básica. A conversão numérica serve para o usuário aprender tipos de conversão e se aprofundar mais sobre, já a criptografia básica serve para criptografar senhas deixando-as ilegíveis tanto para o utilizador quanto para o sistema, mas, é um método reversível.

## •	Como pensaram a usabilidade e acessibilidade?

O objetivo dessa simulação de phishing é alertar as pessoas a nunca colocar dados pessoais em sites que parecem ser de empresas grandes sem olhar direito o URL e o caminho. Sobre acessibilidade, isso também é muito importante, porque geralmente sites de golpistas quase nunca tem uma acessibilidade boa. Eles são mal feitos, confusos, difíceis de ler e não seguem nenhum padrão. Isso acaba atrapalhando ainda mais quem usa leitor de tela ou precisa de uma navegação mais clara, e deixa essas pessoas ainda mais vulneráveis ao golpe.
